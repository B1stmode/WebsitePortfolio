Rocket Dodge

A precision-based escape game where control, timing, and momentum decide whether you make it out — or die buried underground.

What is Rocket Dodge?

You pilot a small evacuation rocket trapped at the lowest level of a man-made cave system on a distant moon. A failed deep-drilling operation has destabilized the entire structure — and the only way out is up.

Read more about the project and my expirience

Each cave layer is a self-contained gameplay level. Survive it, reach the checkpoint platform, and ascend deeper toward the surface.

There is no combat.
There is no safety net.
Just you, physics, and increasingly hostile terrain.

Core Gameplay Loop

  • Navigate tight cave spaces using rotation + thrust
  • Dodge moving mechanical obstacles (pistons, pipes)
  • Avoid static hazards (collapsed rock, debris, structural supports)
  • Reach the landing pad checkpoint at the end of each level
  • Automatically ascend to the next cave layer
  • Difficulty increases through:
    • Obstacle density
    • Placement precision
    • Timing pressure
  • Final level breaks out onto the surface — escape achieved

Controls

  • Rotation: ← / → or A / D
  • Thrust: Space or W

Controls are deliberately tight and unforgiving. Mistakes are on the player — not the game.

Lore Background

The cave system was originally a mining and drilling site. A catastrophic failure at the lowest level triggered structural collapse across all layers.

You control a small rocket tasked with evacuating remaining workers. The crew is never shown on-screen — the focus stays on escape, not exposition.

Technical Overview

  • Engine: Unity 6.1
  • Perspective: 3D environment with 2D movement plane
  • Art Style: Low-poly, modular assets
  • Physics-driven movement
  • Progressive difficulty scaling

What Came From the Course

  • Basic player controller (arrow keys + space)
  • Core 3D assets and models
  • Sound effects and basic FX
  • Fundamental level structure

What I Added Extra

  • Alternative control scheme (W / A / D)
  • Death counter
  • Main menu with settings and level selection
  • Additional levels
  • End-game scene with credits
  • Post-processing improvements
  • Pause menu (currently in progress)

What I Learned

  • Thinking in systems and logic instead of raw syntax
  • Breaking complex problems into smaller solvable parts
  • Translating gameplay ideas into code structure
  • Working effectively with Unity Editor tools and workflows

Rocket Dodge was a turning-point project where I stopped copying code and started thinking like a developer.

Bohunice: Exclusion

Personal solo project — independently developed from scratch

A short first-person atmospheric game set in the immediate aftermath of a nuclear catastrophe in 1977 Czechoslovakia, focused on exploration, tension, and environmental storytelling.

Built outside of school and without following a course or tutorial series. All environments, assets, and systems are created by me.

Read more about the project and development

Project Overview

Bohunice: Exclusion is a solo-developed game project created as a portfolio piece and technical showcase. The goal is to build a cohesive, believable experience with a strong sense of place, using limited scope and carefully controlled mechanics.

Core Experience

  • First-person exploration with guided but non-linear paths
  • Heavy focus on atmosphere, lighting, and sound design
  • Nighttime setting with limited visibility and flashlight use
  • Environmental hazards (radiation, anomalies, unstable terrain)
  • Minimal HUD, information conveyed through the environment
  • Short, self-contained narrative experience

Setting & Narrative Context

The game takes place a few hours after a classified incident at the Jaslovské Bohunice nuclear power plant in January 1977. Due to the nature of the experiment, the facility was only partially guarded, leading to a rapid but incomplete response.

The surrounding village was abandoned almost immediately, while a nearby town was evacuated under false pretenses. The player, a conscript stationed at the plant, regains consciousness after the explosion and is ordered to leave the exclusion zone while carrying a classified containment object.

Technical Overview

  • Engine: Unity
  • Perspective: First-person
  • Visual Style: Low-poly / early-3D-inspired
  • Lighting: Night / early morning, fog, post-processing
  • Level Design: Unity Terrain + custom modular props

Asset Creation

  • All environment models created by me in Blender
  • All textures created by me in GIMP
  • No premade environment asset packs used

Current Development Status

  • Greyboxed playable level with defined progression
  • Core atmosphere established (lighting, fog, post-processing)
  • Foundational gameplay systems implemented
  • Level layout and traversal flow under iteration
  • Visual and narrative polish planned next

Skills Demonstrated

  • Solo project planning and scoping
  • Environment and level design
  • Custom asset creation
  • Gameplay system prototyping
  • Historically grounded world-building

Roll a Dice

Simple web project simulating dice rolls using HTML, CSS, and JavaScript.

What is Roll a Dice?

A web project simulating dice rolls using HTML, CSS, and JavaScript. You can roll one or more dice, view the result instantly, and track all previous rolls in a history list.

Read more about this project

This project demonstrates basic interactivity and DOM manipulation in JavaScript. The UI is responsive and visually clear, making the roll history easy to read and manage.

Core Functionality

  • Roll one or two dice with a click of a button
  • Animated dice roll effect for visual feedback
  • Each dice has its independent roll history
  • "Delete History" button clears all previous rolls
  • Responsive layout for desktop and mobile

Technical Overview

  • Languages: HTML, CSS, JavaScript
  • Interactive buttons and DOM updates
  • Animation controlled via CSS and JS
  • Dynamic roll history list creation and deletion

What Came From the Tutorial

  • Single dice with roll animation triggered by a button
  • Automatic display of dice result in a history list

What I Added Extra

  • Second dice with its own independent history
  • "Delete History" button to clear roll data
  • Custom CSS styling for a cleaner look
  • Responsive layout improvements

What I Learned

  • DOM manipulation: accessing and updating HTML elements dynamically
  • Event handling with buttons and user input
  • JavaScript basics: variables, functions, loops, and conditionals
  • Structuring a small interactive web project

Roll a Dice was my first small JavaScript project where I extended a tutorial by adding meaningful features and improving the user interface. It helped me understand how to manipulate the DOM and structure interactive web applications.
